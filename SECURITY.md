# Security Policy

## Supported Versions

Security fixes are applied to the latest development line of DevForge CLI.

## Reporting A Vulnerability

- Do not open a public GitHub issue for security vulnerabilities.
- Report the issue privately through GitHub security reporting if enabled for the repository.
- If private reporting is not available yet, contact the maintainer through GitHub at `Ali-dev11` and include:
  - a clear description of the issue
  - reproduction steps
  - impact assessment
  - any suggested remediation

## Response Expectations

- Initial acknowledgment: as soon as practical
- Status updates: shared when investigation or remediation meaningfully changes
- Public disclosure: after a fix or mitigation is available whenever possible
